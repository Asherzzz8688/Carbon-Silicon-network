# Code of Conduct

## Purpose

This project exists to support serious collaboration on human rights, AI, safety, governance, and shared abundance.

The subject matter is high-stakes and may involve strong disagreement. Disagreement is welcome. Harassment, intimidation, and bad-faith disruption are not.

## Expected behavior

Participants are expected to:

- treat others with respect;
- argue against ideas rather than attacking people;
- state uncertainty when appropriate;
- welcome serious critique;
- distinguish facts, values, and proposals;
- avoid inflammatory generalizations about groups;
- give credit for others' work;
- respect privacy and consent;
- follow the safety policy;
- help maintain a constructive public commons.

## Unacceptable behavior

Unacceptable behavior includes:

- harassment, threats, or intimidation;
- personal attacks or sustained hostile behavior;
- doxxing or publishing private information without consent;
- discriminatory abuse;
- sexual harassment;
- spam or repeated off-topic disruption;
- impersonation;
- coordinated manipulation of discussion;
- knowingly submitting plagiarized or misrepresented work;
- encouraging real-world harm;
- publishing dangerous technical details in violation of `SECURITY.md`.

## Scope

This code of conduct applies to:

- this repository;
- GitHub Issues;
- GitHub Pull Requests;
- GitHub Discussions;
- project communication channels;
- project events or working groups;
- public behavior that directly affects the project community.

## Enforcement

Maintainers may take action when behavior violates this code of conduct.

Possible actions include:

- reminder or warning;
- request to edit or remove content;
- temporary discussion lock;
- temporary suspension from project spaces;
- removal from contributor roles;
- ban from the project;
- escalation to platform moderators when necessary.

Enforcement should be proportionate, documented when appropriate, and focused on protecting the community and the integrity of the project.

## Reporting

To report a concern, contact the maintainers through the reporting channel listed in `SECURITY.md` or the project's public maintainer contact once one is established.

If the concern involves a maintainer, report it to another maintainer or steward.

Reports should include:

- what happened;
- where it happened;
- who was involved;
- any relevant links or screenshots;
- whether there is an immediate safety concern.

## Good-faith participation

The project will not punish respectful disagreement, minority viewpoints, or high-quality critique.

The project may moderate behavior that makes collaboration impossible, even when the participant claims to be defending an important idea.

## Principle

A community that cannot handle disagreement cannot govern the future.

A community that tolerates abuse cannot protect human dignity.

This project must do both: preserve disagreement and reject abuse.
